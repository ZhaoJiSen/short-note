## 为什么用 TypeScript

TypeScript 是 JavaScript 的超集，核心价值是把一部分运行时错误提前到编译期。

典型收益：
- 在编辑器阶段发现类型错误，而不是上线后报错
- 函数入参与返回值更清晰，团队协作成本更低
- 重构时更安全，改动影响范围可追踪

> [!NOTE]
> TypeScript 只存在于开发和编译阶段，最终运行的是 JavaScript。

## 安装与运行

建议优先使用项目内安装（而不是全局安装）。

```bash
pnpm add -D typescript
pnpm add -D ts-node
```

初始化配置：

```bash
pnpm exec tsc --init
```

编译和运行：

```bash
# 编译单文件
pnpm exec tsc ./src/index.ts

# 直接运行 TS（开发场景常用）
pnpm exec ts-node ./src/index.ts
```

## tsconfig.json 核心配置

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "strict": true,
    "noImplicitAny": true,
    "strictNullChecks": true,
    "skipLibCheck": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  },
  "include": ["src"]
}
```

### 常见选项说明

| 选项 | 含义 | 常见建议 |
| --- | --- | --- |
| `target` | 编译目标 JS 版本 | 现代项目常用 `ES2020+` |
| `module` | 输出模块格式 | 前端工程一般 `ESNext` |
| `moduleResolution` | 模块解析策略 | Vite/Nuxt 常用 `Bundler`，Node 场景常用 `NodeNext` |
| `strict` | 开启严格模式总开关 | 建议开启 |
| `noImplicitAny` | 禁止隐式 `any` | 建议开启 |
| `strictNullChecks` | 区分 `null/undefined` | 建议开启 |
| `skipLibCheck` | 跳过依赖库类型检查 | 大多数项目可开启提升速度 |

## 环境类型补充

如果你在 Node 环境下开发，需要补充 Node 类型：

```bash
pnpm add -D @types/node
```

如果是浏览器项目，确保 `lib` 包含 `DOM`：

```json
{
  "compilerOptions": {
    "lib": ["ES2022", "DOM", "DOM.Iterable"]
  }
}
```
