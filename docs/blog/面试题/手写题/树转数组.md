:::details 面试题

手写树转数组，把树形结构拍平成列表，并保留每个节点的父级关系

:::

## 面试官视角

这题常见于菜单、权限、组织架构：

1. 会不会 DFS 或 BFS
2. 是否保留 `parentId`、`level` 等信息
3. 是否会污染原始树节点
4. 能否反向实现数组转树
5. 对递归爆栈是否有概念

## 示例数据

```js
const tree = [
  {
    id: 1,
    name: 'A',
    children: [
      { id: 2, name: 'B' },
      {
        id: 3,
        name: 'C',
        children: [
          { id: 4, name: 'D' }
        ]
      }
    ]
  }
]
```

期望结果：

```js
[
  { id: 1, name: 'A', parentId: null, level: 0 },
  { id: 2, name: 'B', parentId: 1, level: 1 },
  { id: 3, name: 'C', parentId: 1, level: 1 },
  { id: 4, name: 'D', parentId: 3, level: 2 }
]
```

## DFS 递归实现

### 核心思路

深度优先遍历树。遍历每个节点时，把当前节点除 `children` 之外的信息放入结果数组，然后递归处理它的子节点

::::steps

1. 准备结果数组 `result`
2. 写一个递归函数，参数包含当前节点列表、父节点 id、层级
3. 遍历当前层节点
4. 取出 `children`，把剩余字段和 `parentId`、`level` 放入结果
5. 如果有子节点，递归处理子节点

::::

```js
function treeToArray(tree) {
  const result = []

  function dfs(nodes, parentId = null, level = 0) {
    for (const node of nodes) {
      const { children = [], ...rest } = node

      result.push({
        ...rest,
        parentId,
        level
      })

      if (children.length > 0) {
        dfs(children, node.id, level + 1)
      }
    }
  }

  dfs(tree)

  return result
}
```

## BFS 迭代实现

如果树很深，递归可能爆栈，可以用队列改成广度优先遍历

```js
function treeToArray(tree) {
  const result = []
  const queue = tree.map((node) => ({
    node,
    parentId: null,
    level: 0
  }))

  while (queue.length > 0) {
    const { node, parentId, level } = queue.shift()
    const { children = [], ...rest } = node

    result.push({
      ...rest,
      parentId,
      level
    })

    children.forEach((child) => {
      queue.push({
        node: child,
        parentId: node.id,
        level: level + 1
      })
    })
  }

  return result
}
```

## 顺手准备：数组转树

面试官经常会反问：数组转树怎么做？

核心是先用 `Map` 建立 `id -> node`，再把每个节点挂到父节点的 `children` 上

```js
function arrayToTree(list) {
  const map = new Map()
  const roots = []

  list.forEach((item) => {
    map.set(item.id, {
      ...item,
      children: []
    })
  })

  list.forEach((item) => {
    const node = map.get(item.id)

    if (item.parentId == null) {
      roots.push(node)
      return
    }

    const parent = map.get(item.parentId)

    if (parent) {
      parent.children.push(node)
    }
  })

  return roots
}
```

## DFS 和 BFS 的区别

| 遍历方式 | 输出顺序 | 适合场景 |
| --- | --- | --- |
| DFS | 先父节点，再一路深入子节点 | 菜单展开、保留树的自然顺序 |
| BFS | 一层一层输出 | 按层级处理、找最近节点 |

## 面试回答模板

树转数组本质是树遍历。我会用 DFS，从根节点开始遍历，每访问一个节点，就把它除 `children` 之外的数据放入结果数组，同时补充 `parentId` 和 `level`。然后递归处理它的子节点。为了不污染原数据，我会用解构把 `children` 拿出来，创建一个新对象放进结果

## 易错点

1. 不要直接修改原树节点，否则业务里可能产生副作用
2. `children` 可能不存在，要给默认值
3. `parentId` 根节点一般用 `null`
4. DFS 递归版顺序更贴近树形展示，BFS 迭代版更适合超深树
5. 数组转树时要先建 `Map`，不要每次找父节点都遍历数组，否则会变成 `O(n^2)`
