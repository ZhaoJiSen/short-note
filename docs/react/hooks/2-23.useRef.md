[+Ref]: 一个跨 render 持久存在、可变但不触发重新渲染的容器
[+命令式]: 通过调用方法直接操作实例或 DOM，而不是通过 state 声明结果
[+Imperative Handle]: 通过 `useImperativeHandle` 暴露给父组件的受控命令式 API

`useRef` 最容易被误解：它不是“小号状态”，而是==不会驱动渲染的持久容器==。这也是它特别适合 DOM、定时器、第三方实例、上一次值的原因。

::: table title="哪些值适合放进 ref" full-width
| 场景 | 适不适合 | 原因 |
| --- | --- | --- |
| DOM 节点、滚动容器、输入框 | 适合 | 需要命令式访问真实节点 |
| 定时器 ID、WebSocket 实例、第三方库实例 | 适合 | 需要跨 render 保存，但不必触发视图更新 |
| 页面展示用的数据 | 不适合 | 这类数据变了通常应该重新 render |
| 需要参与 UI 渲染的表单值 | 不适合 | 更适合 `useState` / `useReducer` |
:::

## 场景一：访问 DOM 或暴露受控能力

React 的 `ref` 最常见的两类需求是：访问真实 DOM，以及向父组件暴露一组有限的命令式能力。

:::code-tabs
@tab SearchInput.jsx
```jsx
import { forwardRef, useImperativeHandle, useRef } from 'react';

const SearchInput = forwardRef(function SearchInput(_, ref) {
  const inputRef = useRef(null);

  useImperativeHandle(ref, () => ({
    focus() {
      inputRef.current?.focus();
    },
    clear() {
      if (inputRef.current) {
        inputRef.current.value = '';
      }
    },
  }));

  return <input ref={inputRef} placeholder="请输入关键词" />;
});

export default SearchInput;
```

@tab App.jsx
```jsx
import { useRef } from 'react';
import SearchInput from './SearchInput';

export default function App() {
  const searchRef = useRef(null);

  return (
    <div>
      <SearchInput ref={searchRef} />

      <button onClick={() => searchRef.current?.focus()}>聚焦</button>
      <button onClick={() => searchRef.current?.clear()}>清空</button>
    </div>
  );
}
```
:::

## 场景二：保存跨 render 的值

`useRef` 不只拿 DOM。它同样适合保存“组件重新 render 时还想继续保留，但又不想触发 render”的数据。

:::code-tabs
@tab SearchLogger.jsx
```jsx
import { useEffect, useRef, useState } from 'react';

export default function SearchLogger() {
  const [keyword, setKeyword] = useState('');
  const prevKeywordRef = useRef('');
  const timerRef = useRef(null);

  useEffect(() => {
    clearTimeout(timerRef.current);

    timerRef.current = setTimeout(() => {
      console.log('上一次关键词：', prevKeywordRef.current);
      console.log('这一次关键词：', keyword);
      prevKeywordRef.current = keyword;
    }, 300);

    return () => clearTimeout(timerRef.current);
  }, [keyword]);

  return (
    <input
      value={keyword}
      onChange={(event) => setKeyword(event.target.value)}
      placeholder="试着连续输入内容"
    />
  );
}
```

@tab App.jsx
```jsx
import SearchLogger from './SearchLogger';

export default function App() {
  return (
    <main>
      <h2>useRef 保存跨 render 值</h2>
      <SearchLogger />
    </main>
  );
}
```
:::

## 原理视角：为什么改 `ref.current` 不会 render

`useRef` 返回的是一个稳定对象：

```js
const refObject = {
  current: initialValue,
};
```

React 在整个组件生命周期里都会尽量复用这个对象。你改的是它的 `current` 字段，而不是提交 state 更新，所以默认不会触发重新 render。

:::details 这和 useState 的区别到底是什么
`useState` 变更后会生成 Update，进入调度和 render 流程；`useRef` 只是同步修改内存里的一个字段。一个负责驱动视图变化，一个负责保存命令式引用，它们解决的根本不是同一类问题。
:::

## 使用边界

:::warning
- `ref` 不是用来替代状态管理的
- 能通过 props/state 声明出来的界面，优先用声明式方案
- 只有在 DOM 操作、焦点控制、滚动定位、实例句柄这类需求里，`ref` 才是更自然的选择
:::

:::collapse
- 访问 DOM 时用 `ref`
- 保存“跨 render 持久存在但不驱动渲染”的值时用 `ref`
- 暴露受控的命令式 API 时，配合 `forwardRef` 和 `useImperativeHandle`
- 需要驱动页面更新的数据，不要偷懒塞进 `ref.current`
:::
