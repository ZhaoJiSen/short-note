[+Reducer]: 接收旧 state 和 action，返回新 state 的纯函数
[+Dispatch]: 触发一次状态变更请求的函数

`useState` 和 `useReducer` 处理的是同一件事：==让函数组件在多次 render 之间记住状态==。差别在于，更新逻辑是散落在组件里，还是集中到 Reducer[+Reducer] 中。

::: table title="两个状态 Hook 的分工" full-width
| Hook | 更适合什么场景 | 优点 | 代价 |
| --- | --- | --- | --- |
| `useState` | 单字段、本地状态、简单交互 | 直接、上手快 | 状态多时更新逻辑容易分散 |
| `useReducer` | 多字段联动、状态流转复杂 | 更新规则集中，可测试性更好 | 需要设计 `action` 与 reducer |
:::

## useState：简单状态优先用它

`useState` 最适合“字段不多、变化直接、离组件很近”的场景。比如计数器、输入框、开关状态、分页页码。

:::code-tabs
@tab Counter.jsx
```jsx
import { useState } from 'react';

export default function Counter() {
  // count 保存当前值，setCount 负责提交更新
  const [count, setCount] = useState(0);
  const [step, setStep] = useState(1);

  function handleIncrease() {
    // 新值依赖旧值时，优先用函数式更新
    setCount((prev) => prev + step);
  }

  function handleReset() {
    setCount(0);
    setStep(1);
  }

  return (
    <section>
      <p>当前计数：{count}</p>

      <label>
        步长：
        <input
          type="number"
          value={step}
          onChange={(event) => setStep(Number(event.target.value) || 1)}
        />
      </label>

      <div style={{ marginTop: 12, display: 'flex', gap: 8 }}>
        <button onClick={handleIncrease}>增加</button>
        <button onClick={handleReset}>重置</button>
      </div>
    </section>
  );
}
```

@tab App.jsx
```jsx
import Counter from './Counter';

export default function App() {
  return (
    <main>
      <h2>useState 示例</h2>
      <Counter />
    </main>
  );
}
```
:::

::: card title="什么时候优先选 useState" icon="material-icon-theme:react"
- 状态字段很少，直接 `setXxx` 就能表达清楚
- 更新逻辑和组件视图强绑定，不需要抽离成独立规则
- 你更关心“改成什么”，而不是“这次是什么动作”
:::

## useReducer：复杂状态迁移更清晰

当一个组件里有多个状态字段，而且这些字段会一起变化时，`useReducer` 往往比多个 `useState` 更稳。它的核心价值是：==把更新规则收拢到一处==。

:::code-tabs
@tab reducer.js
```js
export const initialState = {
  keyword: '',
  loading: false,
  todos: [],
};

export function todoReducer(state, action) {
  switch (action.type) {
    case 'keyword/change':
      return {
        ...state,
        keyword: action.payload,
      };

    case 'todos/load-start':
      return {
        ...state,
        loading: true,
      };

    case 'todos/load-success':
      return {
        ...state,
        loading: false,
        todos: action.payload,
      };

    default:
      return state;
  }
}
```

@tab TodoPanel.jsx
```jsx
import { useEffect, useReducer } from 'react';
import { initialState, todoReducer } from './reducer';

export default function TodoPanel() {
  const [state, dispatch] = useReducer(todoReducer, initialState);

  useEffect(() => {
    async function loadTodos() {
      dispatch({ type: 'todos/load-start' });

      // 这里用 Promise 模拟异步请求
      const data = await Promise.resolve([
        { id: 1, title: '整理 Hooks 笔记' },
        { id: 2, title: '补充源码流程图' },
        { id: 3, title: '回顾 React Router' },
      ]);

      dispatch({ type: 'todos/load-success', payload: data });
    }

    loadTodos();
  }, []);

  const visibleTodos = state.todos.filter((item) =>
    item.title.includes(state.keyword)
  );

  return (
    <section>
      <input
        value={state.keyword}
        onChange={(event) =>
          dispatch({
            type: 'keyword/change',
            payload: event.target.value,
          })
        }
        placeholder="按标题搜索"
      />

      {state.loading ? (
        <p>加载中...</p>
      ) : (
        <ul>
          {visibleTodos.map((todo) => (
            <li key={todo.id}>{todo.title}</li>
          ))}
        </ul>
      )}
    </section>
  );
}
```

@tab App.jsx
```jsx
import TodoPanel from './TodoPanel';

export default function App() {
  return (
    <main>
      <h2>useReducer 示例</h2>
      <TodoPanel />
    </main>
  );
}
```

## 怎么选：不是谁更高级，而是谁更适合

::: table title="选型建议" full-width
| 问题 | 更推荐 |
| --- | --- |
| 只是一个开关、一个计数、一个输入框 | `useState` |
| 多个字段会一起变化 | `useReducer` |
| 希望把状态更新逻辑提出来单测 | `useReducer` |
| 只是局部交互，不想额外设计 `action` | `useState` |
:::

:::details 为什么 `setCount(count + 1)` 有时不如函数式更新稳
React 的状态更新不是“立刻改变量”，而是提交一条更新请求。连续多次依赖旧值计算时，函数式更新 `setCount((prev) => prev + 1)` 会更安全，因为 React 会把真正的上一个状态作为 `prev` 传进来。
:::

## 原理视角：React 怎么把状态挂到组件上

函数组件每次 render 都会重新执行，但 React 会把 Hook 数据挂到当前 Fiber 上。`useState` / `useReducer` 在底层都会创建 Hook 节点，并把更新放进队列里。

```js
function mountState(initialState) {
  const hook = mountWorkInProgressHook();

  // Hook 节点上会保存当前 state
  hook.memoizedState = hook.baseState = initialState;

  const queue = {
    pending: null,
    dispatch: null,
  };

  hook.queue = queue;

  // dispatch 闭包里会记住当前 Fiber 和 queue
  const dispatch = (queue.dispatch = dispatchSetState.bind(
    null,
    currentlyRenderingFiber,
    queue
  ));

  return [hook.memoizedState, dispatch];
}
```

```js
const update = {
  lane,      // 本次更新的优先级
  action,    // setState 传入的值，或 reducer 要处理的 action
  next: null,
};
```

> [!NOTE]
> `useState` 可以理解成“内置了一个最简单 reducer 的 `useReducer`”。所以两者在底层更新模型上非常接近，只是开发时写法不同。

:::collapse
- 日常开发里优先关心状态是不是集中、是不是易读
- 复杂状态不要用一堆零散 `setXxx` 硬拼
- 依赖旧值计算新值时，优先用函数式更新
- reducer 一定要返回新对象，不要原地修改旧 state
:::
