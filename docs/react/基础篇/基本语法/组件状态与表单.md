## 受控表单与非受控表单

React 表单可以分成两类：受控组件和非受控组件。

::: table title="表单模式对比" full-width

| 模式 | 特点 | 适合场景 |
| --- | --- | --- |
| 受控组件 | 表单值由 React state 驱动 | 常规业务表单、校验、联动 |
| 非受控组件 | 表单值保留在 DOM，自行用 `ref` 读取 | 文件上传、快速接第三方 DOM 库 |
:::

### 先看一个“状态驱动输入框”的可视化演示

这个 demo 不是 React 代码，它只是用最小例子说明“输入框值和页面展示应该由同一份状态驱动”。

::: demo

```html
<!doctype html>
<html lang="zh-CN">
  <body>
    <label>
      用户名：
      <input id="name" />
    </label>
    <p id="preview">预览：</p>

    <script>
      const state = { name: '' };
      const input = document.getElementById('name');
      const preview = document.getElementById('preview');

      function render() {
        input.value = state.name;
        preview.textContent = '预览：' + state.name;
      }

      input.addEventListener('input', (event) => {
        // 单一数据源：先改状态，再统一刷新视图
        state.name = event.target.value.trimStart();
        render();
      });

      render();
    </script>
  </body>
</html>
```

:::

### 受控表单

```jsx
import { useState } from 'react';

export default function LoginForm() {
  const [form, setForm] = useState({
    username: '',
    password: '',
  });

  function handleChange(event) {
    const { name, value } = event.target;

    setForm((prev) => ({
      ...prev,
      [name]: value,
    }));
  }

  function handleSubmit(event) {
    event.preventDefault();
    console.log('提交数据：', form);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="username"
        value={form.username}
        onChange={handleChange}
        placeholder="用户名"
      />
      <input
        name="password"
        type="password"
        value={form.password}
        onChange={handleChange}
        placeholder="密码"
      />
      <button type="submit">提交</button>
    </form>
  );
}
```

> [!NOTE]
> 这里用到了 `useState`，但重点不是 Hook 本身，而是“表单值由 React 状态统一驱动”这件事。

### 非受控表单

```jsx
import { useRef } from 'react';

export default function UploadForm() {
  const fileRef = useRef(null);

  function handleReadFile() {
    const file = fileRef.current?.files?.[0];
    console.log('当前文件：', file);
  }

  return (
    <div>
      {/* 文件输入框天然更适合非受控方式 */}
      <input ref={fileRef} type="file" />
      <button onClick={handleReadFile}>读取文件</button>
    </div>
  );
}
```

> [!NOTE]
> 非受控表单的关键不在于“少写代码”，而在于数据源留在 DOM 中。`useRef` 的细节和边界，已经统一放到 [useRef](/react/t5qj4s1y/)。

:::collapse

- 普通输入框、文本域、下拉框，优先用受控组件
- 文件上传、第三方富文本、历史老库接入，非受控更省事
- 如果一个表单项需要联动别的表单项、做格式化、实时校验，基本都应该受控

:::
