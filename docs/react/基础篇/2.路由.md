[+SPA]: Single Page Application，单页应用
[+History 模式]: 使用 History API 改写地址栏，不带 `#`
[+Hash 模式]: 使用 `location.hash` 管理路由，地址里会出现 `#`

前端路由解决的不是“请求哪个 HTML 文件”，而是：==当地址变化时，当前页面应该展示哪个组件==。

## 路由是什么

在传统多页应用中，路由更多由服务端控制；而在 `SPA` 中，浏览器通常只加载一次页面骨架，之后由前端根据 URL 决定显示哪个视图模块。

::: table title="两种常见路由模式" full-width
| 模式 | 优点 | 注意点 |
| --- | --- | --- |
| `BrowserRouter` | URL 更自然，像真实页面路径 | 服务端要做 fallback 配置 |
| `HashRouter` | 部署简单，不依赖服务端重写 | URL 带 `#`，可读性稍差 |
:::

:::details 什么时候优先用 HashRouter
如果站点部署在无法配置重写规则的静态服务器上，或者是一些老系统嵌入页，`HashRouter` 会省很多环境问题。但只要服务端可控，优先还是 `BrowserRouter`。
:::

## React Router 常见 API

::: table title="React Router v6 常用能力" full-width
| 分类 | API | 作用 |
| --- | --- | --- |
| 路由容器 | `BrowserRouter` / `HashRouter` | 提供路由上下文 |
| 路由声明 | `Routes`、`Route`、`useRoutes` | 定义匹配规则 |
| 导航组件 | `Link`、`NavLink`、`Navigate` | 声明式跳转 |
| 占位组件 | `Outlet` | 子路由渲染位置 |
| Hooks | `useNavigate`、`useParams`、`useLocation` | 编程式跳转、读取参数与位置 |
:::

## 一个完整的嵌套路由示例

下面的写法和课程中的 `useRoutes` 思路一致，也更适合把路由配置集中管理。

:::code-tabs
@tab main.jsx
```jsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import { BrowserRouter } from 'react-router-dom';
import App from './App';

ReactDOM.createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
);
```

@tab App.jsx
```jsx
import { useRoutes } from 'react-router-dom';
import {
  Navigate,
  NavLink,
  Outlet,
  useLocation,
  useParams,
} from 'react-router-dom';

function Layout() {
  return (
    <div>
      <nav style={{ display: 'flex', gap: 12 }}>
        <NavLink to="/home">首页</NavLink>
        <NavLink to="/users/1001">用户详情</NavLink>
        <NavLink to="/about/profile">关于页</NavLink>
      </nav>
      <hr />
      {/* 子路由匹配后会渲染到这里 */}
      <Outlet />
    </div>
  );
}

function Home() {
  return <h2>Home Page</h2>;
}

function UserDetail() {
  const { id } = useParams();
  const location = useLocation();

  return (
    <section>
      <h2>User Detail</h2>
      <p>动态参数 id：{id}</p>
      <p>上一个页面传来的 state：{location.state?.from ?? '无'}</p>
    </section>
  );
}

function AboutLayout() {
  return (
    <div>
      <h2>About</h2>
      <Outlet />
    </div>
  );
}

function Profile() {
  return <p>Profile Page</p>;
}

function Team() {
  return <p>Team Page</p>;
}

export default function App() {
  return useRoutes([
    {
      path: '/',
      element: <Layout />,
      children: [
        {
          index: true,
          element: <Navigate to="/home" replace />,
        },
        {
          path: 'home',
          element: <Home />,
        },
        {
          path: 'users/:id',
          element: <UserDetail />,
        },
        {
          path: 'about',
          element: <AboutLayout />,
          children: [
            {
              index: true,
              element: <Navigate to="profile" replace />,
            },
            {
              path: 'profile',
              element: <Profile />,
            },
            {
              path: 'team',
              element: <Team />,
            },
          ],
        },
      ],
    },
  ]);
}
```
:::

## 常见导航方式

```jsx
import { Link, useNavigate } from 'react-router-dom';

export default function Actions() {
  const navigate = useNavigate();

  return (
    <div>
      {/* 声明式导航 */}
      <Link to="/users/1001" state={{ from: 'home' }}>
        去详情页
      </Link>

      {/* 编程式导航 */}
      <button onClick={() => navigate('/about/team')}>
        跳到团队页
      </button>
    </div>
  );
}
```

## 动态参数、查询参数、state 的区别

::: table title="页面间传值方式" full-width
| 方式 | 例子 | 适用场景 |
| --- | --- | --- |
| 动态参数 | `/users/:id` | 资源标识、详情页 |
| 查询参数 | `?keyword=react` | 筛选、分页、搜索 |
| `location.state` | `navigate('/x', { state })` | 临时上下文，不要求刷新保留 |
:::

:::warning
- 刷新页面后，`location.state` 可能丢失，所以它不适合存核心业务数据。
- 列表页跳转详情页时，如果详情页必须依赖 `id`，就把 `id` 放在路径参数里，而不是只塞进 `state`。
:::

## 路由使用中的几个细节

:::collapse
- 嵌套路由一定要记得在父组件中放 `Outlet`
- `Navigate` 常用于默认重定向或权限守卫失败后的跳转
- `NavLink` 比 `Link` 多了“当前激活态”能力，适合侧边栏和顶部导航
- `useRoutes` 更适合中大型项目，把路由表集中化后更容易维护
:::
