调用栈（Call Stack）是 JavaScript 引擎管理“函数调用关系”的核心结构。
它遵循 `LIFO`（后进先出）：最后入栈的执行上下文，最先出栈。

## 先用一个直觉模型理解栈

你可以把它想成“单端可操作的容器”：

- 函数被调用：入栈
- 函数执行结束：出栈

![栈示意图](https://cdn.nlark.com/yuque/0/2023/png/28960762/1700803317363-882ba1ed-7846-4c77-87fe-bfabbe0e9340.png)

## JavaScript 为什么需要调用栈

因为 JavaScript 在同一线程上执行，同一时刻只能运行一个执行上下文。
调用栈用于确定“当前正在执行谁”，以及“执行结束后应该回到谁”。

## 用代码跟踪一次完整的入栈/出栈

```js
var a = 2

function add(b, c) {
  return b + c
}

function addAll(b, c) {
  var d = 10
  var result = add(b, c)
  return a + result + d
}

addAll(3, 6)
```

## 如何在浏览器里看调用栈

在 DevTools 的 `Sources` 面板打断点，运行到断点时，右侧 `Call Stack` 会显示当前调用链。

![查看函数调用关系](https://cdn.nlark.com/yuque/0/2023/png/28960762/1701054307930-98b474c0-0018-4c1c-b1fe-5eefd87bc281.png)

一般你会看到：

1. 栈底是全局入口（anonymous / script）。
2. 中间是外层调用函数。
3. 栈顶是当前正在执行的函数。

## 栈溢出（Stack Overflow）

调用栈有容量上限。递归没有终止条件时，会不断创建新执行上下文，最终触发栈溢出。

```js
function division(a, b) {
  return division(a, b)
}

console.log(division(1, 2))
```

常见报错：`RangeError: Maximum call stack size exceeded`

![栈溢出错误](https://cdn.nlark.com/yuque/0/2023/png/28960762/1701054656601-fbc39305-a870-437b-a233-6db8ac8ef859.png)

## 小结

1. 调用栈管理的是执行上下文，不是源代码文本。
2. 入栈/出栈顺序决定了函数调用与返回顺序。
3. 读懂调用栈是定位复杂 Bug 和性能问题的基础能力。
