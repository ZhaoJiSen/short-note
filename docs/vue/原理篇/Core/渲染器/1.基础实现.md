Vue 中的渲染器负责将虚拟 DOM 转换为真实 DOM，并将其渲染到页面中。其本质就是用 JavaScript 对象来描述一个 DOM 结构，然后通过一系列的操作将其转换为真实的 DOM 元素

## render 函数的基本实现

::::: steps

1. `render` 函数的基本设计

   `render` 函数就是用于把一个虚拟节点渲染到一个真实容器中，所以 `render` 至少要有两个参数：这一次要渲染的虚拟节点 `vnode`，以及真实的挂载容器 `container`

   ```ts
   function render(vnode, container) {}
   ```

   函数有了，接下来要考虑它的功能实现

   :::info 实现思路
   - 在首次渲染时：容器中还没有旧的 `vnode`，因此这个时候需要做的是挂载，也就是调用 `mountElement` 函数
   - 再次渲染时：容器中已经有了旧的 `vnode`，因此这个时候需要做的是比较新旧 `vnode` 并更新，也就是调用 `patch` 函数
   - 卸载：新的 `vnode` 不存在，但容器里还有旧内容，这时候要把旧内容移除
   :::

   :::code-tabs

   @tab renderer.ts

   ```ts
   function render(vnode, container) {
     if (vnode) {
        patch(container._vnode, vnode, container);
     } else {
        // 此时表明旧的虚拟 DOM 存在，但新的虚拟 DOM 不存在，为 unmount 操作
        if (container._vnode) {
          container.innerHTML = "";
        }
     }

     // 将每次的 vnode 存储到 container 对象上，以便下次更新时作为旧的 vnode 进行比较
     container._vnode = vnode;
   }
   ```

   :::

2. `createRenderer` 函数的设计

   在 `render` 函数的基本实现中，已经不可避免的将一些其他函数的调用写在了 `render` 中。这也表明他们之间是有耦合关系的，所以可以通过工厂函数 `createRenderer` 将逻辑进行整合

   ```ts
   function createRenderer() {
     function render(vnode, container) {}

     function patch() {}

     function unmount() {}

     function mountElement() {}

     return {
       patch,
       render,
       unmount,
       mountElement,
     };
   }
   ```

3. 平台依赖性的拆分

   在 Vue 源码中，渲染器的设计还考虑了平台依赖性的问题。因为 Vue 不仅可以运行在浏览器环境中，还可以运行在其他平台（如服务器端、原生应用等）中，所以将 DOM 操作与渲染逻辑进行了整体的拆分

   渲染逻辑放入 `runtime-core` 中，平台相关实现放入 `runtime-dom` 中。这样一来，渲染器就可以通过传入不同的 "平台能力" 来适配不同的运行环境，而不需要修改核心的渲染逻辑

   在现阶段的实现中，先试用一个全局对象 `options` 来代替 `runtime-dom` 包中的内容

   :::code-tabs
   @tab renderer.ts

   ```ts
   const options = {
     createElement(tag) {
       return document.createElement(tag)
     },
     setElementText(el, text) {
       el.textContent = text
     },
     insert(el, parent, anchor = null) {
       parent.insertBefore(el, anchor)
     }
   }


   function createRenderer(options) {
     const { createElement, insert, setElementText } = options

     function render() {}
     function patch() {}
     function unmount() {}
     function mountElement() {}

     return {
       render,
       patch,
       unmount,
       mountElement
     }
   }


   const renderer = createRenderer(options);
   ```

   :::

## 挂载的基本实现

::::: steps

1. `mountElement` 函数的基本设计

   前面在 `render` 里已经区分出了首次渲染和更新，而在首次渲染时，真正要做的事情其实就是把当前的 `vnode` 挂到容器里

   这一步对应的就是 `mountElement` 函数，它的作用很直接：接收一个元素类型的 `vnode`，然后把它变成真实 DOM，再插入到容器中

   ```ts
   const vnode = {
     type: "h1",
     children: "hello world",
   };
   ```

   先只看最简单的情况，如果 `children` 是字符串，那么这一步只需要创建元素、设置文本、插入容器就够了

   :::tip 关键点

   `mountElement` 这一层先只做“把一个元素节点挂上去”这件事。也就是说，当前先不处理更新、不处理卸载，甚至也先不展开属性和事件，只把最基础的元素创建流程跑通

   :::

   :::code-tabs

   @tab renderer.ts

   ```ts
   function mountElement(vnode, container) {
     const el = createElement(vnode.type);

     if (typeof vnode.children === "string") {
       setElementText(el, vnode.children);
     }

     insert(el, container);
   }
   ```

   :::

2. 挂载子节点

   只处理字符串子节点当然不够，因为一个元素下面还可能继续挂其他元素

   这时候 `children` 的值就不再是字符串，而是一个数组。数组里的每一项其实也都是一个 `vnode`，所以这里就不能再直接设置文本了，而是要把这些子节点继续交给 `patch` 去处理

   ```ts
   const vnode = {
     type: "h1",
     children: [
       {
         type: "p",
         children: "Hello World",
       },
     ],
   };
   ```

   :::tip 关键点

   这里没有直接在 `mountElement` 里递归调用 `mountElement(child, el)`，而是继续走 `patch(null, child, el)`。原因很简单：对子节点来说，这依然是一轮正常的挂载流程，只不过旧节点不存在，所以第一个参数传的是 `null`

   :::

   :::code-tabs

   @tab renderer.ts

   ```ts
   function mountElement(vnode, container) {
     const el = createElement(vnode.type);

     if (typeof vnode.children === "string") {
       setElementText(el, vnode.children);
     } else if (Array.isArray(vnode.children)) {
       vnode.children.forEach((child) => {
         patch(null, child, el);
       });
     }

     insert(el, container);
   }
   ```

   :::

3. 挂载元素属性

   子节点挂上去以后，下一步就该处理元素本身的属性了。这里可以先给 `vnode` 增加一个 `props` 字段，用来描述元素上的属性

   ```ts
   const vnode = {
     type: "h1",
     props: {
       id: "foo",
     },
     children: [
       {
         type: "p",
         children: "Hello World",
       },
     ],
   };
   ```

   这一层最直接的做法，就是在挂载元素时遍历 `vnode.props`，然后把这些值设置到真实元素上

   :::tip 关键点

   这一阶段先不要把属性设置想得太复杂，先记住最基本的一层就够了：`props` 是虚拟节点里用来描述元素属性的，挂载时要把它们同步到真实元素上

   :::

   :::code-tabs

   @tab renderer.ts

   ```ts
   function mountElement(vnode, container) {
     const el = createElement(vnode.type);

     if (typeof vnode.children === "string") {
       setElementText(el, vnode.children);
     } else if (Array.isArray(vnode.children)) {
       vnode.children.forEach((child) => {
         patch(null, child, el);
       });
     }

     if (vnode.props) {
       for (const key in vnode.props) {
         const value = vnode.props[key];
         el.setAttribute(key, value);
       }
     }

     insert(el, container);
   }
   ```

   :::

4. 属性为什么不能一律用 `setAttribute`

   到这里代码已经能跑，但这里马上会碰到一个更细的问题：元素属性并不总是适合统一走 `setAttribute`

   比如：

   - HTML 标签里写的是 `class`
   - DOM Properties 上对应的是 `className`

   还有像 `value`、`checked`、`disabled` 这种属性，直接走 `setAttribute` 和直接设置 DOM Properties，行为也并不完全一样

   所以这里要先记住一个结论：**HTML Attributes 和 DOM Properties 不是完全等价的**

   :::info 这里先记住一个原则

   `HTML Attributes` 更像是在设置 DOM 的初始状态，而有些场景下，真正应该操作的是元素对象本身的 `DOM Properties`

   :::

5. 区分 `DOM Properties` 和 `Attributes`

   最典型的情况就是布尔属性

   ```ts
   const vnode = {
     type: "button",
     props: {
       disabled: "",
     },
     children: "hello world",
   };
   ```

   如果直接写：

   ```ts
   el.setAttribute("disabled", "");
   ```

   浏览器会把它理解成按钮处于禁用状态。但如果你传的是 `false`，`setAttribute` 这层语义就会变得不够自然。相对来说，直接设置：

   ```ts
   el.disabled = false;
   ```

   结果就更符合直觉

   所以这里就需要补一个简单判断：如果某个 key 本身就存在于元素对象上，那么优先按 `DOM Properties` 去处理；否则再退回 `setAttribute`

   :::tip 关键点

   这一层不是为了追求“属性设置方式统一”，而是为了尽量让虚拟节点里的值和真实 DOM 的行为保持一致。尤其是像布尔属性这种情况，只走 `setAttribute` 很容易让结果和直觉不一致

   :::

   :::code-tabs

   @tab renderer.ts

   ```ts
   function mountElement(vnode, container) {
     const el = createElement(vnode.type);

     if (typeof vnode.children === "string") {
       setElementText(el, vnode.children);
     } else if (Array.isArray(vnode.children)) {
       vnode.children.forEach((child) => {
         patch(null, child, el);
       });
     }

     if (vnode.props) {
       for (const key in vnode.props) {
         const value = vnode.props[key];

         if (key in el) {
           const type = typeof el[key];

           if (type === "boolean" && value === "") {
             el[key] = true;
           } else {
             el[key] = value;
           }
         } else {
           el.setAttribute(key, value);
         }
       }
     }

     insert(el, container);
   }
   ```

   :::

6. 用 `shouldSetAsProps` 收口判断

   即使加上了 `key in el` 这层判断，也还是会遇到例外情况。比如：

   ```html
   <form id="form1"></form>
   <input form="form1" />
   ```

   这里 `form` 虽然能在元素对象上找到对应的属性，但它是只读的，这种时候就不应该直接走 `el[key] = value`，而是应该退回 `setAttribute`

   所以更稳一点的做法，是把“这个属性到底该不该作为 DOM Properties 设置”单独封装成一个函数

   :::tip 关键点

   `shouldSetAsProps` 的作用，就是把“哪些属性应该走 DOM Properties，哪些应该走 Attributes”这层判断单独收起来。这样 `mountElement` 里就只负责调用，而不用把各种特殊情况都摊在循环里

   :::

   :::code-tabs

   @tab renderer.ts

   ```ts
   function shouldSetAsProps(el, key, value) {
     if (key === "form" && el.tagName === "INPUT") {
       return false;
     }

     return key in el;
   }

   function mountElement(vnode, container) {
     const el = createElement(vnode.type);

     if (typeof vnode.children === "string") {
       setElementText(el, vnode.children);
     } else if (Array.isArray(vnode.children)) {
       vnode.children.forEach((child) => {
         patch(null, child, el);
       });
     }

     if (vnode.props) {
       for (const key in vnode.props) {
         const value = vnode.props[key];

         if (shouldSetAsProps(el, key, value)) {
           const type = typeof el[key];
           if (type === "boolean" && value === "") {
             el[key] = true;
           } else {
             el[key] = value;
           }
         } else {
           el.setAttribute(key, value);
         }
       }
     }

     insert(el, container);
   }
   ```

   :::
