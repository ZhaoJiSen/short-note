[+依赖收集]: 这个过程也被称为 "依赖收集"。通过数据，找到依赖它的函数
[+依赖追踪]: 这个过程也被称为 "依赖追踪"。通过函数执行过程，反向记录它用到了哪些数据
[+副作用函数]: 副作用函数在执行过程中，==会影响外部环境，或者依赖外部状态=={.important}。比如 `console.log`、DOM 操作、网络请求、读取全局变量等

Vue 的响应式，本质上是在 ==**函数** 和 **它运行时读取过的数据** 之间建立联系=={.important}。如果一个函数在执行时用到了某些数据，那么这些数据变化后，这个函数往往也应该重新执行

:::details 为什么不是所有函数都自动参与响应式
Vue 并不会强制所有函数都和数据建立关系，而是只处理那些需要响应更新的函数。需要依赖数据的函数，就建立关联；不需要依赖数据的函数，就不建立关联。这样做的好处是：

- 避免无意义的追踪
- 只在真正需要更新的地方触发重新执行
- 把控制权交给开发者，而不是把所有逻辑都塞进响应式系统
:::

## 核心流程

核心是让函数如何知晓依赖了哪些数据，因此实现响应式的关键点是：==数据要能被拦截，函数执行时要能被记录=={.important}。Vue 会先把数据做成响应式对象，在函数运行期间，记录它访问了哪些属性，最后当属性变化时，反向找到相关函数并重新执行

:::table full-width

| 阶段 | Vue 在做什么 | 目的 |
| --- | --- | --- |
| 1. 标记数据 | 把普通数据变成响应式数据 | 让读取和修改都可被感知 |
| 2. 执行函数 | 运行副作用函数[+副作用函数] | 收集它实际访问了哪些数据 |
| 3. 建立关系 | 把函数与它读取过的数据关联起来[+依赖收集] | 形成依赖关系 |
| 4. 触发更新 | 数据发生变化时，找到相关函数重新执行[+依赖追踪] | 完成自动更新 |

:::

换个角度看，就是这条链路：`函数执行` -> `读取数据` -> `建立依赖` -> `数据变化` -> `重新执行函数`

::::details 实现差异

:::table full-width

| 版本 | 实现方式 | 说明 |
| --- | --- | --- |
| Vue2 | `Object.defineProperty` | 通过属性劫持追踪读写 |
| Vue3 | `Proxy` | 通过代理对象统一拦截，更灵活，也更完整 |

:::

::::

## 实现流程

### 基本实现

::::steps

1. 先准备三个最基本的角色

   原始数据对象 `obj`、一个副作用函数 `effect`、一个依赖容器 `buckets`

   :::details 代码

   ```js
   const obj = {
     name: 'John',
     age: 20,
   };

   const buckets = new Set();

   function effect() {
     app.innerHTML = proxy.name;
   }
   ```

   :::

   这里的 `effect` 就是 "数据变化后需要重新执行的函数"，`buckets` 则用来存放这些函数

2. 用 `Proxy` 包一层，让读取和修改都能被拦截

   只有拦截住这两个动作，后面才有机会做依赖收集和触发更新

   :::details 代码

   ```js
   const proxy = new Proxy(obj, {
     get(target, key, receiver) {},
     set(target, key, value, receiver) {},
   });
   ```

   :::

3. 在 `get` 中做最简单的依赖收集

   当 `effect` 执行到 `proxy.name` 时，会触发 `get`。这时就把 `effect` 放进 `buckets`，表示这个函数依赖了当前读取的数据

   :::details 代码

   ```js
   get: (target, key, receiver) => {
     buckets.add(effect);
     return Reflect.get(target, key, receiver);
   }
   ```

   :::

4. 手动先执行一次 `effect`，完成首次渲染和首次收集

   这一步非常关键，因为如果 `effect` 从来没执行过，就不会触发 `get`，也就不会建立依赖关系

   :::details 代码

   ```js
   effect();
   ```

   :::

5. 在 `set` 中做最简单的触发更新

   当执行 `proxy.name = 'Jane'` 时，会触发 `set`。`set` 先把值真正写回原对象，遍历 `buckets`，把里面收集到的函数重新执行一遍

   这也就完成了：`数据变化` -> `找到相关函数` -> `重新执行函数` -> `页面重新渲染`

   :::details 代码

   ```js
   set: (target, key, value, receiver) => {
     const result = Reflect.set(target, key, value, receiver);
     buckets.forEach((fn) => fn());
     return result;
   }
   ```

   :::

6. 整条链路

   :::table full-width

   | 阶段 | 触发点 | 发生的事 |
   | --- | --- | --- |
   | 首次执行 | `effect()` | 触发副作用函数，开始读取响应式数据 |
   | 依赖收集 | `get` | 把 `effect` 收进 `buckets` |
   | 修改数据 | `proxy.name = 'Jane'` | 触发 `set` |
   | 派发更新 | `buckets.forEach(fn => fn())` | 把收集到的副作用函数重新执行 |
   | 更新视图 | `app.innerHTML = proxy.name` | DOM 显示最新值 |

   :::

::::
