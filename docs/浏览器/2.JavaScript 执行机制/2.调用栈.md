:::details 栈结构

栈结构是一种 "后进先出" 的结构：最后入栈的执行上下文，最先出栈

![栈示意图](https://cdn.jsdelivr.net/gh/ZhaoJiSen/note-images@master/browser/assets/js-execution/1700803317363-882ba1ed-7846-4c77-87fe-bfabbe0e9340.png)

:::

## 调用栈

[+调用栈]: 执行上下文栈

在 JavaScript 执行过程中，可能会存在多个执行上下文。此时就需要 JavaScript 引擎对上下文进行统一管理调用，而栈结构恰好符合要求

:::details 为什么需要调用栈

因为 JavaScript 在同一线程上执行，同一时刻只能运行一个执行上下文。调用栈用于确定 "当前正在执行谁"，以及 "执行结束后应该回到谁"

:::

JavaScript 引擎正是利用栈结构这种后进先出的结构来 管理执行上下文的。==不同的执行上下文创建好后，JS 引擎便将其压入栈中，等待执行完毕后，JS 引擎便将其从栈中弹出==。因此这种用来管理执行上下文的栈就被称为调用栈[+调用栈]

### 调用栈解析

::::steps

1. 示例代码

   ```js
   var a = 2

   function add(b, c) {
     return b + c
   }

   function addAll(b, c) {
     var d = 10
     var result = add(b, c)
     return a + result + d
   }

   addAll(3, 6)
   ```

2. 创建并压入全局执行上下文

   创建全局执行上下文，并将其压入栈底。此时变量 `a`、函数 `add` 以及 `addAll` 全部都会在变量环境中保存一个与之同名的对象名称

   :::details 图示

   ![全局执行上下文压栈 =750x300](https://cdn.jsdelivr.net/gh/ZhaoJiSen/note-images@master/browser/assets/js-execution/1700807304878-acf1ee50-581e-486f-8db3-e30b32593044.png)

   :::

3. 执行全局代码

   全局执行上下文执行压栈后，执行全局代码。执行 `a = 2`

   :::details 图示

   ![赋值操作改变执行上下文中的值](https://cdn.jsdelivr.net/gh/ZhaoJiSen/note-images@master/browser/assets/js-execution/1700806811228-f000b7cc-b907-4d22-8c87-f9ae6c497b1a.png)

   :::

4. 调用全局函数 `addAll`

   JavaScrip 引擎从变量环境中取出函数代码，进行编译，创建执行上下文压入栈中。同样的函数执行上下文同样会创建变量环境和词法环境，然后收集函数内的变量声明和函数声明

   :::details 图示

   ![执行 addAll 函数前一刻的调用栈](https://cdn.jsdelivr.net/gh/ZhaoJiSen/note-images@master/browser/assets/js-execution/1700806981071-9e37f08f-6c2b-44ec-adb3-ac86a7693821.png)

   :::

5. 执行 `addAll`

   函数执行上下文压入栈后，执行函数代码。将 `d` 由 `undefined` 赋值为 10，当执行到函数内部的函数 `add` 时，同样会为其创建执行上下文，并将其压入调用栈

   `add` 函数执行上下文会执行与之前函数上下文同样的操作

   :::details 图示
   ![add 函数执行前一刻时的调用栈](https://cdn.jsdelivr.net/gh/ZhaoJiSen/note-images@master/browser/assets/js-execution/1700807677874-a5ac53d3-4b7c-4bb3-a469-5656784cc4dc.png)
   :::

6. `add` 执行完毕

   内层 add 函数执行完毕，该函数的执行上下文就会从栈顶弹出，若存在返回值则将返回值返回。返回结果后，控制权回到 `addAll`

   :::details

   ![add 函数执行结束时的调用栈](https://cdn.jsdelivr.net/gh/ZhaoJiSen/note-images@master/browser/assets/js-execution/1700807770596-7be7b8fb-3152-41de-9ac2-7cc0ff8c1c18.png)

   :::

7. `addAll` 执行完毕

   紧接着外层函数 `addAll` 执行完毕并返回，外层函数的执行上下文也会从栈顶弹出，最终只剩全局执行上下文

   :::details
   ![addAll 函数执行结束时的调用栈](https://cdn.jsdelivr.net/gh/ZhaoJiSen/note-images@master/browser/assets/js-execution/1700807846284-3177c6d7-f580-4713-90a0-e9f3ae80cb31.png)
   :::

::::

### 查看调用栈

在 DevTools 的 `Sources` 面板打断点，运行到断点时，右侧 `Call Stack` 会显示当前调用链

![查看函数调用关系](https://cdn.jsdelivr.net/gh/ZhaoJiSen/note-images@master/browser/assets/js-execution/1701054307930-98b474c0-0018-4c1c-b1fe-5eefd87bc281.png)

:::note 一般会看到：

1. 栈底是全局入口（anonymous / script）
2. 中间是外层调用函数
3. 栈顶是当前正在执行的函数
:::
